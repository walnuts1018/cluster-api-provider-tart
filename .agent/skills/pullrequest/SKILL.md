PRの説明は、変更内容を単に羅列するのではなく、変更の目的や意図を簡潔に表現するようにしてください。
また、PRのタイトルは、変更内容がひと目で分かるような分かりやすいものにしてください。
関連するIssueがある場合は、`Closes #1` や `Fixes #2` のように説明に含めて自動でクローズされるようにしてください。

## レビューについて

- GitHub Copilotが自動的にレビュワーとしてアサインされます。
- PR作成後、Copilotのレビューが終わるまで待機し、必要に応じて指摘された点を修正してください。
  - 必ず正しいレビューをしてくるとは限りません。そのレビューが間違っており、コードの修正が必要ない場合は、レビューに対してコメントを残し、なぜ修正が必要ないのかを説明してください。
